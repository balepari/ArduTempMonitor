I choose this board for the number of its I/O and more program capacity. So if I want to develop more stuff I don\textquotesingle{}t have to deal with I/O or memory. Also is the only board that I found a junction board with screw terminals so is more stable to connect cable and other stuff to screw terminal instead pinning them into a molex pin.\hypertarget{md___users_paride__workarea__arduino__ardu_temp_monitor__r_e_a_d_m_e_autotoc_md2}{}\doxysubsection{What it does?}\label{md___users_paride__workarea__arduino__ardu_temp_monitor__r_e_a_d_m_e_autotoc_md2}

\begin{DoxyItemize}
\item Read temperature from D\+S18\+B20
\item Write on L\+CD with date and time
\item Send same data via U\+DP packet to a remote server for storing, analysis and graphing (the server has a listening process written in Python that get udp packets and write to mysql server. See ......... for the repos.)
\end{DoxyItemize}\hypertarget{md___users_paride__workarea__arduino__ardu_temp_monitor__r_e_a_d_m_e_autotoc_md3}{}\doxysubsection{Feature}\label{md___users_paride__workarea__arduino__ardu_temp_monitor__r_e_a_d_m_e_autotoc_md3}

\begin{DoxyItemize}
\item uses a Dallas D\+S18\+B20 as temperature probe
\item uses a Dallas D\+S3121 as R\+TC to have clock with date and time
\item uses a Keyestudio K\+S0156 Ethernet shield which mount W\+I\+Znet W5100 chip (I prefer cabled ethernet connection instead of Wi\+Fi)
\item uses an I2C L\+CD 20x4 from Quimat just for wiring simplifications and more speedy coding
\end{DoxyItemize}
